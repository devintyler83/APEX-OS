# APEX OS State Snapshot

APEX OS is a deterministic draft operating system that turns 16 ranking sources, measurables, and film context into one board you can defend after the draft, not just before it. It is season-scoped (2026, `season_id=1`), migration-safe, and designed for full auditability: every source, score, tag, and override is versioned and traceable. The engine builds a weighted consensus spine, runs deterministic APEX scoring against archetype and measurables libraries, and then layers divergence, tags, and decision cards on top. Public entry points are `https://apexsignal.app` for the marketing front door and `https://apexsignal.streamlit.app/` for the live app runtime.

Streamlit LFS Boot Fix (draftos/db/connect.py):
  Root cause identified: data/edge/draftos.sqlite is tracked in Git LFS (.gitattributes).
  Streamlit Cloud does not fetch LFS objects. The "DB file" on Streamlit Cloud is the 134-byte
  LFS pointer text file. sqlite3.connect() opens it silently as an empty DB. PATHS.db.exists()
  returns True on the pointer. Queries then fail: "no such table: prospect_board_snapshots".
  Fix: added _validate_db(path) to connect.py:
    Guard 1: reads first 16 bytes; raises RuntimeError with LFS fix instructions if header ≠
    "SQLite format 3\x00". Catches LFS pointer before any query is attempted.
    Guard 2: queries sqlite_master; raises RuntimeError listing missing tables if any of
    {prospect_board_snapshots, prospects, apex_scores, sources, seasons, models} absent.
  _validate_db() is called from connect(create_ok=False) only — no impact on pipeline writes.
  Permanent fix still required: move DB out of Git LFS (see Next Milestone).
